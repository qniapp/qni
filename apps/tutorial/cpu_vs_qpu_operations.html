--- layout: page section: QPU 命令その 1 title: CPU 命令との違い description: 量子コンピュータ版 論理演算 ---

QPU 命令は、CPU で実行される AND や NOT, OR といった命令 (論理演算) の量子ビット版です。 QPU 命令と CPU 命令には、次の大きな違いがあります。

  • QPU 命令は重ね合わせ状態に作用できます。

    CPU 論理演算は 0 または 1 のみを入力・出力します。 0 と 1 は古典 1 ビットで表せる値のすべてです。

    一方で QPU 命令は量子ビットを入出力します。 量子ビットは 0 と 1 だけでなく、0 と 1 の重ね合わせ状態を持つことができます。 QPU 命令は 0 と 1 の確率や、量子ビットの位相にも作用します。

    {% include minus.svg class="h-6 w-6 fill-current text-neutral-400" %} QPU 命令 {% include arrow_right.svg class="h-6 w-6 fill-current text-neutral-400" %}

  • QPU 命令の大部分は、逆方向の演算 (逆演算) を持ちます。

    演算を適用した後に逆演算を適用すると、量子ビットを元の状態に戻すことができます (後にいくつかの QPU 命令と逆演算を具体的に見ます)。 一方で CPU 論理演算はそのほとんどが非可逆です。たとえば、AND 演算の出力 1 ビット (0 または 1) から入力の 2 ビット (00, 01, 10, 11) を知ることはできません。

逆演算は量子プログラミングでひんぱんに使うテクニックの 1 つであるアンコンピュテーションにとって重要です。 アンコンピュテーションとは、実行中にぐちゃぐちゃになった量子ビットの状態を元の初期状態に戻して再利用する方法で、少ない量子ビット数でやりくりするための量子版ガベージ・コレクションとも言えるものです。 後にそのやり方を見ていきますが、量子プログラミング言語にはまだガベージ・コレクションのような高級な機能はないので、プログラマが明示的に実装してやる必要があります。
